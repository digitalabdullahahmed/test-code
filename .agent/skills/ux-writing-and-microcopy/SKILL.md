# UX Writing And Microcopy

Use this skill whenever frontend work needs stronger product language, microcopy, or user-facing text decisions.

## Goal

Make interfaces easier to understand, trust, and complete by using precise, helpful UI copy.

## Core rules

- write for task completion, not decoration
- prefer clear verbs over vague placeholders
- make the next action obvious
- keep terminology consistent with product and backend domain language
- reduce anxiety in error, destructive, and auth-sensitive flows

## Required concerns

- field labels
- button text
- helper text
- validation messages
- server and error messages
- empty states
- success confirmations
- destructive confirmations

## Review checklist

Reject or fix:
- vague action labels
- generic empty states with no next step
- error messages that do not help recovery
- inconsistent terminology across the same feature
- stylish copy that reduces clarity in critical flows
