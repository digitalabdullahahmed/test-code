# Design Hand Off And UI Governance

Use this skill whenever the task affects UI consistency, design-to-code fidelity, or governance across many frontend contributors.

## Core rules

- design decisions should be encoded into reusable systems
- visual drift should be corrected early
- implementation should preserve product intent, not approximate it casually

## Required concerns

- UI ownership rules
- design token governance
- component usage consistency
- handoff interpretation discipline
- visual review expectations

## Review checklist

Reject or fix:
- growing visual drift across modules
- repeated one-off components that should be shared
- unclear ownership of design decisions
- low-fidelity implementation of important product surfaces
