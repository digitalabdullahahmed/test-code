# Forms Validation UX

Use this skill whenever the task affects forms, submissions, user input, or mutation-driven UI.

## Core rules

- forms must be accessible
- validation must be explicit
- submission states must be visible
- failure recovery must preserve user trust

## Required concerns

- field labels and error messages
- schema validation
- pending and disabled states
- server error mapping
- success feedback
- preserving input on failure where appropriate

## Review checklist

Reject or fix:
- unlabeled fields
- hidden submission states
- validation only on the client
- poor server error display
- forms that lose user input unnecessarily
