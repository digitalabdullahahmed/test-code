# Frontend Code Review Refactor Standards

Use this skill whenever the task is a frontend review, refactor, or quality-improvement effort.

## Core rules

- find correctness, UX, performance, and maintainability risks first
- review structure before cosmetic detail
- improve naming and boundaries where they reduce future cost
- comments should preserve reasoning, not narrate JSX

## Focus areas

- component boundaries
- state ownership
- rendering behavior
- accessibility regressions
- naming quality
- maintainability drift

## Review checklist

Focus on:
- bugs and UX regressions
- missing test coverage
- weak naming
- oversized components
- hidden state complexity
- comment noise or missing reasoning comments where needed
