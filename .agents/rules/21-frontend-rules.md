# Frontend Rules

- keep `app/` route-focused and thin
- keep feature logic in `modules/`
- prefer Server Components by default
- keep `use client` boundaries narrow
- prefer derived state over redundant or contradictory local state
- centralize API access patterns
- design for accessibility, responsive behavior, performance, and clear UX feedback
- avoid generic AI-looking output
- use deliberate visual direction
- require clear primary actions, understandable copy, and explicit async states
