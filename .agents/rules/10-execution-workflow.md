# Execution Workflow

Default execution sequence:

1. identify the real task and affected surfaces
2. choose the relevant skills
3. research if facts may be outdated or unstable
4. inspect local code before changing it
5. implement the smallest coherent change set
6. verify with typecheck, tests, build, or focused checks
7. fix failures before closing
8. review follow-on impact on related files, contracts, tests, docs, routes, metadata, and configs
9. summarize outcome, verification, and remaining risks

Do not stop after planning when the user expects implementation.
