# Content And Microcopy Rules

- labels, button text, and helper text must be concrete, concise, and action-oriented
- avoid vague interface copy such as `Submit`, `Continue`, `Done`, or `Click here` when a clearer action name is possible
- error messages should explain what happened, what the user can do next, and what input needs correction when applicable
- empty states should explain the situation and present the next meaningful action
- success states should confirm the result without forcing the user to infer what changed
- keep terminology stable across routes, forms, dialogs, and backend domain language
- do not let decorative language reduce clarity in critical workflows
