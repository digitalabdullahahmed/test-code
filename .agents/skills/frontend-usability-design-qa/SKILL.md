# Frontend Usability Design QA

Use this skill when frontend work should be reviewed like a strong design and UX QA pass.

## Goal

Catch the issues that still make a page feel unprofessional even when the code technically works.

## Review dimensions

- visual hierarchy and readability
- mobile and cross-device behavior
- interaction clarity
- loading, empty, success, and error states
- accessibility and keyboard behavior
- copy quality and terminology consistency
- spacing, density, and alignment
- data integrity and integration cues

## QA workflow

1. identify the primary user task on the screen
2. verify the primary action is obvious
3. inspect hierarchy, spacing, and readability
4. inspect form and async feedback states
5. inspect mobile, tablet, and desktop behavior
6. inspect accessibility, focus, and keyboard flow
7. inspect copy, terminology, and error recovery
8. list issues by severity before summary

## Review checklist

Reject or fix:
- screens with weak primary action hierarchy
- visually polished pages with confusing flows
- broken or awkward mobile behavior
- missing async feedback states
- inconsistent terminology or misleading copy
- layouts that technically render but feel dense, fragile, or unfinished
