# Storybook Component Documentation

Use this skill whenever the task affects Storybook, component docs, usage examples, or shared UI discoverability.

## Core rules

- shared UI should be discoverable
- examples should reflect real usage
- documentation should help contributors use components correctly

## Required concerns

- stories for core primitives
- state coverage
- usage examples
- prop clarity
- visual review support

## Review checklist

Reject or fix:
- shared components with no usage examples
- stories that only show trivial happy states
- docs that drift from actual component behavior
- Storybook treated as decoration rather than engineering support
