# Animation Motion Interaction Patterns

Use this skill whenever the task affects animation, motion, or interaction choreography.

## Core rules

- motion should support comprehension, not distract
- keep motion consistent with the design system
- prefer a few meaningful transitions over constant animation noise
- respect reduced-motion preferences

## Required concerns

- page or section transitions
- loading choreography
- hover and pressed feedback
- motion tokens or timing consistency
- reduced-motion handling

## Review checklist

Reject or fix:
- animation that slows down core workflows
- inconsistent motion behavior
- no reduced-motion support
- motion added as decoration without UX value
