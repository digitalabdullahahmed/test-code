# Frontend Design

Use this skill whenever the user wants a frontend that feels intentionally designed, not merely functional.

## Primary outcomes

Build interfaces that are:
- production-grade
- visually striking
- cohesive
- memorable
- aligned to product context

## Design thinking

Before coding, define:
- purpose
- audience
- user goals and primary tasks
- tone
- technical constraints
- one unforgettable quality

Commit to a clear conceptual direction. Examples:
- brutally minimal
- maximalist
- editorial
- retro-futuristic
- luxury refined
- playful
- organic
- industrial
- brutalist
- art deco geometric

## Aesthetic rules

- avoid generic AI aesthetics
- avoid default SaaS layouts unless the existing system requires them
- avoid generic fonts like Arial, Inter, Roboto, and system stacks unless preserving an existing design language
- avoid timid color distribution and bland spacing
- use strong visual hierarchy
- use memorable typography
- use motion intentionally
- use background treatment and atmospheric detail where it supports the concept

## Frontend aesthetics guidance

### Typography

- pair a distinctive display font with a refined body font when appropriate
- use typography as a structural tool, not just decoration

### Color and theme

- define CSS variables
- commit to dominant colors and controlled accents
- avoid generic purple gradients and generic dark mode by default

### Motion

- use a few meaningful moments: entry sequences, hover states, scroll reveals, loading choreography
- respect reduced-motion preferences

### Spatial composition

- use asymmetry, overlap, negative space, or controlled density deliberately
- break the grid only on purpose

### Backgrounds and detail

- use texture, gradients, shapes, grain, decorative borders, or depth when they fit the chosen direction
- build atmosphere instead of flat default backgrounds

## Implementation rules

- code must remain maintainable
- accessibility is still required
- mobile and desktop must both work well
- UX clarity is required, not optional polish
- design choices must support task completion, not only visual novelty
- visual boldness must not destroy clarity

## Review checklist

Reject or fix:
- generic AI-looking layout
- default-looking typography
- no clear design direction
- no clear primary action or weak hierarchy
- random motion with no system
- decorative choices that hurt usability
