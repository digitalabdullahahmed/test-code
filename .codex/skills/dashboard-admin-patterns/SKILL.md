# Dashboard Admin Patterns

Use this skill whenever the task affects internal dashboards, operational panels, or admin-heavy interfaces.

## Core rules

- optimize for clarity and speed of operation
- dense does not mean chaotic
- admin users need strong information hierarchy and predictable actions
- tables, filters, and details should work together coherently

## Required concerns

- dashboard layout hierarchy
- navigation clarity
- table and details interaction
- bulk actions
- destructive-action safety
- audit-friendly UX where relevant

## Review checklist

Reject or fix:
- overloaded screens with weak hierarchy
- destructive actions too easy to trigger
- poor alignment between list, detail, and edit flows
- weak empty and loading states in admin surfaces
