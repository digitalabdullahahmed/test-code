# Team Delivery Checklist

Use this checklist for tasks that span multiple layers.

## Intake

- what is the actual user outcome?
- which layers are affected?
- is the task production, scaffold, or exploratory?
- what assumptions are safe and what would be risky?

## Ownership

- architecture and boundaries
- backend and contract work
- frontend and UX work
- shared types and packages
- tests and verification
- docs and release notes

## Integration

- contract aligned?
- auth and permissions aligned?
- error handling aligned?
- loading and empty states aligned?
- docs and tests updated?
- rollout or migration implications reviewed?
