# Real Implementation Rules

- do not present placeholders, fake behavior, or TODO-only code as completed work
- avoid mock returns, hard-coded fallbacks, or demo data in real execution paths unless the user explicitly asked for scaffolding
- prefer a smaller real implementation over a wider fake surface
- if a temporary stub is unavoidable, mark it clearly and call it out explicitly in the final summary
- remove debug-only shortcuts, dead toggles, and temporary wiring before claiming completion unless they are intentionally part of the task
